Alliance OS Research Projects

Developing a new OS is never easy, and even if the big picture and targets are clear, the road to go is long and dangerous. In order to have a clearer understanding of some particular tecniques that will be used in the /development branch, some experimentation is both good and functional. Also here are built some tools useful to support the development of the mainline.

Basically the coding rules and directory naming conventions used in the Alliance OS are followed, even if in a less stricter way. It is possible to use some code from /development branch, but in general it is pretty independent from the mainline in term of code and configuration, so we avoid to pollute a research environment with unnecessary complexity and we avoid to mess up the mainline itself. It is also possible to use all the facilities offered by the hosting development platform, Linux or another decent UNIX, so UNIX libraries can be used here in the code (the only exception in the development line is inside testing module code). In general the /research branch is a place where we put Alliance OS standard code wherever this does not imply unnecessary complexity or unnatural choices, helping the solution of specific implementation problems in Alliance coding style, and leaving the rest not necessarily compliant. It is important though not to go too far in this direction, since lazyness is not always a good choice.

Each research project has its own life cycle: some research projects are only an ad-interim solution to some specific problems, while others are more geared to be used as debugging tools in the Alliance OS life. Research projects also may be created raher quickly. Below are listed the currently defined ones, with a small description and current state (Alpha if it is in the early stage of development, Beta if it is a mature research tool).
